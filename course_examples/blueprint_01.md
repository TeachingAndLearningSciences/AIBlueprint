# AI - Education Blueprint – Behavior-Driven Testing in Software Quality

| Section | Content |
|--------|---------|
| **1. General Lecture Description** | **ACM Category:** Software Engineering (SE) <br> **Knowledge Unit:** SE-Requirements <br> **Lecture Description:** The lecture introduces Behavior-Driven Development (BDD) and Behavior-Driven Testing (BDT) as a collaborative and specification-based testing approach that improves communication between developers, testers, and stakeholders. It covers BDT concepts, tools (like Cucumber), and the role of AI in generating and evaluating BDT scenarios. Students will explore how AI models can assist in writing feature files, test automation code, and identifying missing test cases through NLP-based requirements analysis. |
| **2. Lecture Contents** | **Learning Objectives:** <br> *Course-related LOs:* <br> - LO1: Describe the principles and process of Behavior-Driven Testing. *(Understand)* <br> - LO2: Apply BDT techniques using tools such as Cucumber and Gherkin syntax. *(Apply)* <br> - LO3: Analyze feature files for completeness and consistency. *(Analyze)* <br> *AI-related LOs:* <br> - LOAI-1: Explain how Gherkin/BDD works, key phases, etc. <br> - LOAI-2: Use AI to draft Gherkin feature files from user stories or requirements. *(Generate)* <br> - LOAI-3: Interpret AI-generated test cases and relate them to intended behaviors. *(Explain)* <br> - LOAI-4: Refine simple BDT scenarios using AI (accuracy, relevance). *(Improve)* <br><br> **Topics Covered:** <br> - Principles of BDT and its support for software quality <br> - Structure and syntax of Gherkin scenarios <br> - Test automation using BDT frameworks (e.g., Cucumber + Selenium/Java/Python) <br> - Using LLMs to generate feature files from natural language requirements <br> - Critical assessment of AI-generated test scenarios |
| **3. Lecture Activities** | **Description:** <br> Students will engage in hands-on exercises using BDT tools and AI-assisted generation of feature files. They’ll work in small teams to translate natural language user stories into testable Gherkin scenarios manually and with AI, then automate these scenarios. <br><br> **Activities / Hands-On:** <br> - Create basic requirements and user stories <br> - Manually write Gherkin scenarios from user stories *(Apply)* <br> - Generate JUnit test stubs from scenarios <br> - Use ChatGPT to create Given/When/Then scenarios *(Generate)* <br> - Use ChatGPT/Copilot in VS Code to generate full features *(Generate)* <br> - Analyze AI-generated scenarios for edge cases and completeness *(Analyze/Evaluate)* <br> - Execute tests using Cucumber and discuss results *(Apply/Evaluate)* <br><br> **Tools / Technologies:** <br> - ChatGPT or GitHub Copilot <br> - Cucumber with Java/Python <br> - Gherkin editors (e.g., Pickles, GherkinLint) <br> - VS Code with BDD extensions |
| **4. Assessment** | **Assessment Methods:** <br> - *Mini Project:* Students receive user stories and produce Gherkin scenarios, automate them with BDT tools, and evaluate AI-generated content. <br> - *Peer Review:* Students critique human-written vs. AI-generated feature files. <br> - *Reflective Quiz:* Students reflect on the strengths/weaknesses of AI-generated BDT. <br><br> **Constructive Alignment:** <br> - Activities align with learning objectives <br> - Projects and reflections promote understanding of both BDT concepts and AI involvement |
| **5. Lecture Reflection** | **Discussion Prompts:** <br> - Can AI tools fully replace manual BDT writing? <br> - What risks are associated with blindly trusting AI-generated test cases? <br><br> **Critical Thinking:** <br> - Examine flawed AI-generated scenarios for incorrect assumptions <br> - Discuss the need for human oversight <br><br> **Reflection:** <br> - Reflect on the experience of using AI in BDT – limitations, benefits, and risks |
