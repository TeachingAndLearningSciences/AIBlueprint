


# AI - Education Blueprint -
# (BDD Lecture - in Software Quality Course)


## 1. General Course and Lecture Description


|   |  |
|---------------------------|------------------------------------------------|
| **ACM Category:**         | Software Engineering (SE)|
| **Knowledge Unit:**       | SE-Requirements  |
| **Course Overview:**       | The software quality assurance course for computer science students provides an overview of core SE principles and practices related various different quality aspects. This includes testing software systems, hazard analysis, and safety assurance for safety-critical systems, as well as process assessment and improvement techniques. |
| **Course Modalities:**       | Lecture+Lab, 90-120min      |
| **Lecture Context:**       | The lecture introduces  Behavior-Driven Development (BDD) and Behavior-Driven Testing (BDT) as a collaborative and specification-based testing approach that improves communication between developers, testers, and stakeholders. It covers BDT concepts, tools (like Cucumber), and the role of AI in generating and evaluating BDT scenarios. Students will explore how AI models can assist in writing feature files, test automation code, and identifying missing test cases through NLP-based requirements analysis.|
| **Keywords/Topics:**       | Requirements <br> Testing <br> Behavior-Driven Development    |

## 2. Lecture Contents  & Learning Objectives





|    | **Lecture-related LO**    | | 
|---------------------------|------------------------------------------------|----|
| LO_C1  | Students are able to understand and describe the core principles and process steps that are part of Behavior-Driven Development and of Behavior-Driven Testing. | [EXPLAIN]|
| LO_C2  | Students are able to use the BDD principles and apply BDD techniques using tools such as Cucumber and the Gherkin language to create BDD features and scenarios. |[APPLY]|
| LO_C3  | Students are able to generate tests for the respective scenarios . |[APPLY]|
| LO_C4  | Students are able to analyze feature files and assess their suitability for testing, their completeness, and consistency. | [EVALUATE]|



|    | **LAI-related LO**    || |
|---------------------------|------------------------------------------------|----|---|
| LO_AI1  |  Students are able to use an AI assistant to query for explanations about the core concepts, phases, and elements of BDD and respective tools. | [EXPLAIN]|[AI-Usage: Explain]|
| LO_AI2  | Students are able to draft Gherkin feature files from user stories or requirements in the given-when-then format  |[APPLY] |[AI-Usage: GENERATE]|
|LO_AI3 | Students are able to interpret AI-generated test cases and relate them to intended behaviors.  |[APPLY]|[AI-Usage: Evalute] |
| LO_AI4  |  Students are able to assess and refine simple  BDD scenarios generated by an AI tool and critically assess their accuracy and relevance, and further refine them when needed.  | [EVALUATE] |[AI-Usage: Refine]|
